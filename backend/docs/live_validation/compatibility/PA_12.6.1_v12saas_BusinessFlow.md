## Findings requiring a decision

- **Parser failures**: none observed — both heuristics (rule `DB('...')` scan, TI CellPut scan) produced plausible, manually-spot-checked edges on real IBM content.
- **TM1py API mismatches**: **Defect #002 (fixed)** — for PA SaaS, TM1py's `_determine_auth_mode()` must see ONLY `base_url` + `user="apikey"` + password; passing `tenant`/`database` as kwargs routes it into the CPD/PA-Proxy branch which demands `cpd_url`. Fix: `build_tm1_kwargs` v12_saas branch constructs the base_url itself. Regression test added (`test_build_tm1_kwargs_v12_saas_mode`).
- **Permission-related discrepancies**: none observed (trial API key has full access; low-privilege key untested).

### Observations (not defects)

1. **Broad "survey" questions exhaust the tool-round budget**: asking the developer agent (5 rounds) to enumerate writes across all 26 processes ran out of rounds mid-investigation and returned partial text (graceful, as designed, but $0.24 for an incomplete answer). Scoped questions work perfectly. Options if this recurs: higher budgets for survey tasks, or persona guidance to prefer graph queries (`get_object_relationships` incoming edges) over per-object iteration.
2. **The agent probes dead connections**: the system prompt lists every org connection; the agent first tried the defunct native "Dev Server" (7s tool error) before using the live SaaS connection. Graceful, but pruning inactive/failing connections from the prompt would save time and tokens.

## Overall assessment

- [x] **Safe to proceed to Release 3 (TM1 Write Platform) planning** — read platform qualified against PA as a Service 12.6.1
- Remaining before broader confidence: an on-prem/native-auth environment (CAM still unsupported), a low-privilege key test, and a larger model for performance data.
